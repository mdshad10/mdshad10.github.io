Smart Basketball-Hoop Scoring Tracker

Overview

Our final project is a smart basketball-hoop scoring tracker where a player has to make as many shots on the basketball hoop in a set period of time. We are using a low-power infrared break-beam sensor which is mounted below the rim; every time the light connection is interrupted by the ball passing through and covering the laser, the microcontroller instantly registers a make, increments an internal score counter, and provides feedback in three ways. For every make that is registered, one of the serial LEDs will light up green for a second, the buzzer will play a sequence of frequencies representing a “Cha-Ching!” sound, and the updated score is displayed on the LCD. These features allow the player to know when they score and appreciate their make. The timer for how long the round will last is chosen by the player. Switch 1 is used to increment the timer by 5 seconds, and if it is held for 2 seconds or longer, it is set to the default 30 seconds for a round. Switch 3 is pressed when the player is satisfied with the amount of time set, and to start the game. Then a 3-second timer runs to allow the player to get ready, and when the buzzer plays a loud high-pitched sound, it signals the start of the round. When the round starts, the timer starts counting down from the time set by the player, and the timer is displayed on the LCD. Once the time expires, the round ends.

Inspiration

Our inspiration came from a smart basketball hoop installed at Tompkins Square Park in New York City (by Huupe), which tracks shots in real time and gives audio-visual feedback. We wanted to recreate a similar experience using our own microcontroller and the embedded-systems techniques we’d learned: interrupt handling, real-time timing, ADC sampling, and hardware control.

Technical Approach

We modeled the game as a finite-state machine (IDLE → PREP → RUN → END), driven by:

  • SysTick interrupt for millisecond-accurate clock, debouncing, and FSM timing.
  • PORTC GPIO interrupts for SW1/SW3 button presses (no polling).
  • ADC-driven break-beam on PTC2 (channel 11); a reading >900 counts triggers check_breakbeam().
  • Buzzer playback via buzzer_play_tone(), which toggles PTC1 at the desired frequency and inherently debounces additional beam breaks.
  • LCD display helpers lcd_show_timer() and lcd_show_score() to update only when values change.

Testing & Debugging

We used a modular, incremental approach—verifying the break-beam, buzzer, LCD, and FSM independently before integration. Key highlights:

  • Calibrated beam threshold (<700/1023) by serial-monitor ADC dumps.
  • Fine-tuned buzzer frequencies for “Cha-Ching!”, start, end, and victory tones.
  • Validated LCD digit mapping and ghost-free updates.
  • Stress-tested timer transitions across all preset durations.
  • Hot-glued beam assembly to resist hoop vibration; taped net aside to prevent false triggers.
