Smart Basketball-Hoop Scoring Tracker

An embedded system that tracks basketball shot accuracy with real-time feedback

Overview

Our final project is an intelligent basketball scoring system that tracks successful shots within a customizable time period. The system utilizes an infrared break-beam sensor mounted beneath the hoop rim to detect successful baskets. When a ball passes through the hoop, interrupting the laser beam, the microcontroller:

  • Increments an internal score counter
  • Activates visual feedback (LED illumination)
  • Generates audio confirmation ("Cha-Ching!" sound)
  • Updates the LCD display with the current score

The game duration is fully customizable by the player. Switch 1 adjusts the round length in 5-second increments (up to 60 seconds), with a long press resetting to a 30-second default. Switch 3 initiates gameplay, triggering a 3-second countdown followed by an audible start signal. Throughout gameplay, the LCD continuously displays both the remaining time and current score.

This project was inspired by commercial smart basketball systems like those from Huupe, which combine shot tracking with real-time feedback to enhance player experience. Our implementation demonstrates core embedded systems concepts including interrupt handling, real-time timing, and hardware control using the FRDM-KL46Z microcontroller.

Technical Implementation

The system operates as a finite-state machine (FSM) with four distinct states:

  1. IDLE: Waiting for player input to configure game duration
  2. PREP: 3-second countdown before gameplay begins
  3. RUN: Active gameplay with real-time scoring
  4. END: Game completion with final score display

Key Technical Features:

  • Precise Timing: SysTick provides millisecond-accurate system clock for debouncing and countdown
  • Interrupt-Driven Input: PORTC interrupts handle button presses without polling
  • Accurate Scoring: ADC-driven break-beam detection with >900 count threshold
  • Multi-Sensory Feedback: Integrated LED, buzzer, and LCD responses
  • Power Efficiency: All peripherals operate on 3.3V rail without level shifting

Development Process

Testing Methodology

We employed an incremental testing strategy to ensure system stability:

  1. Breakbeam Sensor: Calibrated ADC thresholds through physical testing (700/1023 threshold)
  2. Audio Feedback: Iteratively refined buzzer tones for distinct game events
  3. Display System: Verified LCD digit conversion and segment positioning
  4. Game Logic: Simulated timer and scoring behavior before hardware integration

Key Insight: The blocking buzzer implementation serendipitously solved a scoring edge case by creating a natural debounce period, preventing multiple score registrations from a single basket.

Challenges & Solutions

  • Mechanical Stability: Secured components with hot glue to prevent vibration-induced false triggers
  • Net Interference: Taped net aside to prevent false beam breaks
  • Power Distribution: Utilized J9 pins for additional 3.3V supply when main pins were occupied
  • Scoring Accuracy: Implemented blocking delay to prevent multiple counts per shot
